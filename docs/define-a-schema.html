cobra.Schema

In general, you will start using Cobra by creating a Schema. Each schema maps to data a web service expects to receive. This can be useful whether you are mapping to data that will reside in a SQL database (like MySQL), a NoSQL database (like MongoDB), or a realtime webservice such as Firebase. Each schema maps to a collection and defines the shape of the documents within that collection.

options

The Schema has a few options that can be passed in during its construction or later when it is invoked. The options are:

  • ignore: Array - Used when you want to skip over a definition if the value is equal to one of the items in the array. For example, wanted to prevent null or empty strings from being included in your model, you would have an array [null, ''].
