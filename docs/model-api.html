Cobra’s model API provides functionality to to register and consume custom SchemaTypes, SchemaHelpers, and models. It also provides some shortcuts to perform “one-off” validations.

model.applySchema

Invokes the schema to applying helpers and validation checks against the definition. This function is asynchronous and returns a promise to handle the results. Cobra uses D.js to handle promises.

Example

model.getName

Getter used to return the model’s name.

Example

model.getSchema

A function used to execute a registered SchemaType stand-alone. Useful when you want to perform some “formatting” without the overhead of creating a Schema.

Example

model.get

Getter used to return a value from a string path.

Example

model.set

Setter used to set a value from a string path. If the path does not exist, it will be created.

Example

model.options

Getter / Setter used to register and generate a model from a Schema.
