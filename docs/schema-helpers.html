Cobra provides several helper functions to help format data.

Using helpers

SchemaHelpers are included as part of the definition. You can use any number of helpers in a definition. Unlike a SchemaType, SchemaHelpers will not throw an error. If the value does not match the required type, the SchemaHelper will do nothing. In the following example, we use the trim and lowercase helpers in our role definition.

 `SchemaHelpers` are called prior to validating the data type.
The order in which you declare `SchemaHelpers` in the definition will be the order in which they are applied.

Schema helpers

Below is a list of available schema helpers that are part of Cobra. Additional schema types are available or can be added (See “Custom schema helpers”)

lowercase

lowercase : Boolean
Creates a setter which calls .toLowerCase() on the value. Expects value to be a String.

uppercase

uppercase : Boolean
Creates a setter which calls .toUpperCase() on the value. Expects value to be a String.

trim

trim : Boolean
Creates a setter which calls .trim() on the value. Expects value to be a String.

min

min : Number
Creates a setter which calls Math.min() on the value. Expects value to be a Number.

max

max : Number
Creates a setter which calls Math.max() on the value. Expects value to be a Number.

round

round : Boolean
Creates a setter which calls Math.round() on the value. Expects value to be a Number.

ceil

ceil : Number
Creates a setter which calls Math.ceil() on the value. Expects value to be a Number.

floor

floor : Number
Creates a setter which calls Math.floor() on the value. Expects value to be a Number.
