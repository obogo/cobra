Cobra’s API provides functionality to to register and consume custom SchemaTypes, SchemaHelpers, and models. It also provides some shortcuts to perform “one-off” validations.

cobra.schemaType

A getter / setter used to create custom SchemaTypes. If the function is undefined, the function will act as a getter. For details, view documentation “Custom schema types”.

cobra.applySchemaType

A function used to execute a registered SchemaType stand-alone. Useful when you want to perform some “formatting” without the overhead of creating a Schema.

cobra.validate

A helper function that can be used to create and validate a value in one step. This can be useful when validating elements in a form, or some other data where an entire schema is not required. validate works just like applySchema, accepting a definition and options when invoked. Unlike model.applySchema, cobra.validate, is synchronous and the result is returned immediatly upon execution.
