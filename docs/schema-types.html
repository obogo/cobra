Schema types

Below is a list of available schema types that are part of Cobra. To see how to add your own SchemaTypes (See “Custom schema types“)

String

Expects value to be a String, If the value is not a String, it will be casted into a String value. The only exception is null. Null will be converted into an empty String.

Number

Expects value to be a numeric, otherwise an error will be thrown. See “Schema validators” to view details on valid Numeric types.

Boolean

Expects value to be a truthy or falsy, otherwise an error will be thrown. See “Schema validators” to view details on valid Boolean types.

Date

Expects value to be a Date type, otherwise an error will be thrown. See “Schema validators” to view details on valid Date types.

Mixed

An “anything goes” SchemaType, its flexibility comes at a trade-off of it being harder to maintain. Mixed is available either through cobra.Schema.Types.Mixed or by passing an empty object literal. The following are equivalent:

Array

Creates an array of SchemaTypes or embedded objects.

Int

Expects value to be an Integer type, otherwise an error will be thrown.

Currency

Expects value to be an Numeric value containing either 0 or 2 decimals places, otherwise an error will be thrown.

Email

Expects value to be a valid Email, otherwise an error will be thrown.

One-off types

Sometimes you want to validate without registering a SchemaType. You can do this two different ways - either throught a function or Regular expression.

Function

Instead of refering to a SchemaType on the “type” property, replace with a function.

Regular Expression

Instead of refering to a SchemaType on the “type” property, replace with a RegExp.
